# Documentation

Use this folder for product, API, and operational notes as the build progresses.

## Docs index

- `docs/architecture.md` — system flow and component overview.
- `docs/integrations.md` — webhook payloads, retry rules, and admin headers.

## What lives here

- `info.md` (repo root) remains the source of requirements; mirror decisions and deeper implementation notes here.
- `docs/integrations.md` should capture webhook endpoints, payload contracts, and third-party dependencies.
- `docs/architecture.md` outlines the system flow, core components, and persistence model.
- `docs/runbooks/` should store operational runbooks (deploys, incident response, data fixes).
